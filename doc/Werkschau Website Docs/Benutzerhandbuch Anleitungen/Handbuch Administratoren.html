Aufbau im Ordner System

Die Inhalte der Werkschau Seite sind im Ordner content organisiert. 

-- content
	-- absolventen    // Unterseite für die Übersicht von Absolventen
	-- location       // Unterseite für die Location Informationen
	-- timetable      // Unterseite für Zeitpläne
	-- speakers       // Unterseite für Speaker
	-- blocks         // Enthält konfigurerte Block(Seiten) für die Nutzung in Templates
	-- buildings      // Enthält die Gebäude Unterseiten
	-- datenschutz    // Standardseite für den Datenschutz
	-- error          // Standard Fehlerseite
	-- home           // Website Root
	-- impressum      // Standardseite für das Impressum
	-- students       // Unterseite für Studenten (enthält alle Studenten, Arbeiten ...)
	-- talks          // Unterseite für Talks / Vorträge
	-- teamworks      // Unterseite für Teamarbeiten

Zurücksetzen der Seiteninhalte

-- content
	-- speakers       // Unterseite für Speaker
	-- blocks         // Enthält konfigurerte Block(Seiten) für die Nutzung in Templates
	-- buildings      // Enthält die Gebäude Unterseiten
	-- students       // Unterseite für Studenten (enthält alle Studenten, Arbeiten ...)
	-- talks          // Unterseite für Talks / Vorträge
	-- teamworks      // Unterseite für Teamarbeiten

Beispiel am Ordner für Speaker

-- /content
	-- /speakers           // Der Überordner muss erhalten bleiben
		-- /unterordner      // Unterordner können gelöscht werden
		-- speakers.txt      // Die Textdatei im Ordner muss erhalten bleiben

Überblick über das Panel

Navigation

Die Inhalte der Seite sind im Administrator Panel verwaltbar. Die einzelnen Anwendungsgebiete werden in Tabs organisiert.

Paneltab: Seiten

Das Tab Seiten enthält alle Unterseiten der Werkschau Website. Diese Seiten können veröffentlicht und im Menü der Werkschau Website angezeigt werden. Sie dienen also der Strukturierung der Inhalte auf erster Ebene.

Als so genannte Standardseiten sind folgende Typen vorhanden:

  • Startseite (mit Auswahl von Blocks für die jeweiligen Inhalte)
  • Einfache Seite - Eine Seite zur Erstellung einfacher Informationsseiten (z. B. Impressum)
  • Sections Page (Block-Seite) - Eine Seite zur Erstellung von Seiteninhalten mit Hilfe von Blocks

Mehr zu Standardseiten:

Paneltab: Blocks

Blocks dienen der Organisation von wiederverwendbaren und konfigurierbaren Inhalten. Blocks können auf Blockseiten eingebunden werden und die Inhalte werden auf der jeweiligen Seite angezeigt.

Vorteil von Blocks ist deren Wiederverwendbarkeit auf diversen Seiten. So kann beispielsweise ein Headerblock für die Startseite und ein Headerblock für Unterseiten angelegt und gesetzt werden.

Anwendungsbeispiel:

Es soll ein Header konfiguriert werden der auf diversen Blockseiten eingebunden werden kann. Hierfür wird ein Headerblock angelegt. Durch die Konfiguration und Auswahl von Bildern wird der Headerblock spezifiziert. Ist der Block nun aktiv, kann dieser auf Blockseiten mit dem Auswahlfeld Headerblock ausgewählt werden und wird hierdurch angezeigt.

Paneltab: Studenten

Im Paneltab für Studenten können diese angelegt, befüllt und veröffentlicht werden.

Eine Studentenseite kann Abschlussarbeiten und Semesterarbeiten enthalten.

Paneltab: Speaker & Talks (teilweise implementiert)

Im Paneltab Speaker und Talks können Speaker mit zugehörigen Arbeiten angelegt werden.

Diese Speaker können wiederum einem Talk zugeordnet werden (noch nicht möglich) um hieraus eine Übersicht über Vortragende und Vorträge erstellen zu können.

Paneltab: Seitendaten

Über das Paneltab Seitendaten werden die globalen Daten zur Seite gepflegt. Hierbei mitunter Adresse und Anschrift, Soziale Links, Mapbox Access Token, Dozenten und Studiengänge.

Paneltab: Infrastruktur (teilweise implementiert)

Das Paneltab Infrastruktur behandelt die Gebäudeverwaltung. Hierbei können Gebäude und Räume mit zugehörigen Daten erfasst werden.

Die Gebäude dienen der Organisation von Ausstellungen und bilden die Grundlage für eine Raum / Gebäudeübersicht.

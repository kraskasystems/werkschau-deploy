Repository Struktur

-- root
	-- dist     // Ordner für die Entwicklung der Seitenstrukturen und Layout
	-- src      // Ordner für die zu verpackenden Ressourcen
	-- util     // utils für Fake-Daten, Helfer für die Webpack Config, nützliche Skripte
	-- www      // kirby Entwicklungsverzeichnis
...
webpack.config.html-build.js  // für die Entwicklung des Website Gerüstes
webpack.config.kirby-build.js // für die Auslieferung in die Kirby Asset Struktur

Dateistruktur Kirby Verzeichnis

-- ...
-- www
	-- assets   // css, fonts, images, js, vendor libraries 
	-- content  // generierte Inhalte durch das Panel
	-- kirby    // kirby core
	-- media    // media cache (kirby generierte thumbnails ect.)
	-- site     // Dateien für die Konfiguration des Systems
	-- util     // Node.js basierte Utils für Fake Daten und Helper

Organisation der Kirby Inhalte

Die Inhalte der Werkschau Seite sind im Ordner content organisiert. 

-- content
	-- absolventen    // Unterseite für die Übersicht von Absolventen
	-- location       // Unterseite für die Location Informationen
	-- timetable      // Unterseite für Zeitpläne
	-- speakers       // Unterseite für Speaker
	-- blocks         // Enthält konfigurerte Block(Seiten) für die Nutzung in Templates
	-- buildings      // Enthält die Gebäude Unterseiten
	-- datenschutz    // Standardseite für den Datenschutz
	-- error          // Standard Fehlerseite
	-- home           // Website Root
	-- impressum      // Standardseite für das Impressum
	-- students       // Unterseite für Studenten (enthält alle Studenten, Arbeiten ...)
	-- talks          // Unterseite für Talks / Vorträge
	-- teamworks      // Unterseite für Teamarbeiten

Zurücksetzen der Seiteninhalte

-- content
	-- speakers       // Unterseite für Speaker
	-- blocks         // Enthält konfigurerte Block(Seiten) für die Nutzung in Templates
	-- buildings      // Enthält die Gebäude Unterseiten
	-- students       // Unterseite für Studenten (enthält alle Studenten, Arbeiten ...)
	-- talks          // Unterseite für Talks / Vorträge
	-- teamworks      // Unterseite für Teamarbeiten

Beispiel am Ordner für Speaker

-- /content
	-- /speakers           // Der Überordner muss erhalten bleiben
		-- /unterordner      // Unterordner können gelöscht werden
		-- speakers.txt      // Die Textdatei im Ordner muss erhalten bleiben

Organisation der Werkschau Funktionalität

Dateien für grundlegende Seiten

  • Homepage (Index von Kirby, kann weitergeleitet werden)
  • Sections Page (dynamische Seiteninhalte mit Blocks)
  • Error Seite
  • Default Seite (Seite für einfache Inhalte mit Markdown-Textfeld)

befinden sich direkt in den Verzeichnissen von site:

-- site
	-- blueprints    // Seitenblueprints für die Konfiguration der Felder im Panel
	-- models        // Models für die Erweiterung des standard Page Models
	-- controllers   // Controller für die Auslagerung von Logik
	-- snippets      // Snippets für wiederverwendbare Code Teile
	-- templates     // Templates für die Generierung der Seiteninhalte
	-- ...           // weitere Ordner die nicht für grundlegende Seiten zuständig sind

Dateien für einzelne Bereiche (Plugins) der Werkschau Website

-- site
	-- plugins    // Verzeichnis für die Plugins und Bereiche der Werkschau Website
		-- ...

Abgebildete Bereiche und Funktionalitäten

Die Werkschau Website wurde in folgende Bereiche unterteilt:

  • Studenten ( Student Plugin ) - zur Organisation der Studenten und Arbeiten
    • Student
    • Abschlussarbeit
    • Semesterarbeit
    • Teamarbeit
  • Speaker und Talks ( Speaker Plugin ) - zur Organisation des Speaker / Talk Systems
    • Speaker
    • Talks
  • Suche ( Search Plugin ) - zur Organisation der Suchfunktionalitäten
    • Suchseite (Volltextsuche und Tag basierte Suche)
    • Suchformular
    • API-Endpunkte für die Suche (Content Representations)
  • Role Panel ( Role Panel ) - zur Konfiguration des Panels nach Benutzerrolle
    • Panelkonfiguration für Administratoren
    • Panelkonfiguration für Studenten
  • Infrastruktur ( Infrastructure Plugin ) - zur Verwaltung von Gebäuden und Räumen
    • Gebäude
  • Blocks ( Block Plugin ) - zur Bereitstellung von konfigurierbaren Blocks

    Grundsätzlicher Gedanke ist es, Seiteninhalte über das Panel konfigurierbar zu machen. Ein Block ist eine Seite im Sinne von Kirby, jedoch nicht selbst aufrufbar.

    Blocks enthalten Informationen, Text und Konfigurationen für die Anzeige von Inhalten und werden über Snippets in das jeweilige Seitentemplate eingebunden. ( Block Plugin )

    Aktuell verfügbare Blocks:

    • Hero (block-hero) - Konfiguration für den Seitenheader
    • Info (block-info) - Konfiguration für Informationen
    • Maps (block-maps) - Konfiguration für Google Maps / Mapbox
    • Students (block-students) - Konfiguration zur Anzeige von Studenten in einem Grid

Aufbau eines Bereiches / Plugins

-- sample-plugin
	-- blueprints    // Seitenblueprints für die Konfiguration der Felder im Panel
	-- models        // Models für die Erweiterung des standard Page Models
	-- controllers   // Controller für die Auslagerung von Logik
	-- snippets      // Snippets für wiederverwendbare Code Teile
	-- templates     // Templates für die Generierung der Seiteninhalte
	-- index.php     // Initialisierungsdatei für das Plugin

Dateien für Ressourcen und Assets (CSS, JS ect.)

Ressourcen und statische Assets sind im Ordner assets hinterlegt.

-- assets
	-- js       // JavaScript Dateien (enthält Unterordner)
	-- css      // CSS Dateien (enthält Unterordner)
	-- img      // Bilddateien
	-- fonts    // Schriftdateien
	-- vendor   // Externe Bibliotheken

Kirby bietet die Funktionalität, CSS und JavaScript Dateien automatisch und basierend auf dem jeweiligen Template Namen zu laden. Die Werkschau Website ist auch hierauf ausgelegt.
